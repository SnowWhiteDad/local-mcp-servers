MCP Stdio External Resources only

Basic stdio communication flow with external resource access for the MCP server processes.

Overview

This diagram illustrates the basic stdio-based communication pattern between an AI agent and MCP servers, showing how local MCP servers can access external resources while maintaining process isolation.

Local Communication

AI agent communicates with MCP servers via stdio (stdin/stdout) for maximum security and process isolation.

External Access

MCP servers can make API calls to external services while the stdio transport remains local-only.

Security Model

No network ports or sockets involved - pure process-to-process communication for enhanced security.
